Cohort AACE Insight — Admin Dashboard

A superadmin view that shows an institution's demo cohort how their students actually performed across the AACE Success Abilities — making one point visible: marks measure recall; ability is distributed. No single child leads everything; every child anchors a strength. This is a design storyboard for sign-off, on sample data.

The approach

  • Two views. A Cohort view (anonymous — Student A…J) that shows how the group varies, and a Per-Student view (identified, one child at a time, for a consented parent/teacher context).
  • Prove it with ability data alone. The "marks ≠ ability" message is shown using the spread of Success Abilities across the cohort — it does not depend on marks we don't hold.
  • Extend, don't rebuild. This deepens two reports that already exist (batch-comparison + student-scores) and reuses the scorecard's own banding (PerformanceBand).

Assumptions (so the numbers are honest)

  • Only reviewed evaluations count; each student shows completeness ("5 of 7 reviewed") — partial cohorts aren't averaged as if complete.
  • Data is sparse — each task measures only a few abilities — so unmeasured cells read "no data", never 0.
  • The ability set is per-challenge (₹1999 ≈ Top-15, but some programs are 16–20; ₹4999 ≈ 62–64) — columns are derived from the selected challenge, not hardcoded.
  • Small cohorts → show bands, not fine 1–N ranks (rank shuffles at n=10 are noise).
  • Cohort = batch; date filter = batches.start_date (units inherit the batch date for manual demos).

Guardrails (children's data · brand)

  • Anonymous by default. A school never sees a named league-table of minors. Names appear only in the single-child consented view.
  • No ranking/labelling of named children to a third party; no off-tenancy export of identified minors. Consistent with our "not to label or rank students" promise.
