Why change it

Today, approving for a child needs a separate guardian email and a 6-digit code the parent has to fetch and type — which blocks the normal Indian family that shares one email. Per the spec, we drop the separate-email + code requirement and replace the check with something stronger and already there: the parent's payment (a real card/UPI charge is a far harder-to-fake adult signal than an email code). The parent gives a clear, named approval; the flow gets shorter, and the record is more defensible.
