Tapping an expired / already-used / tampered link never dead-ends or 403s — it lands here with a friendly banner and a one-tap way forward (email pre-filled if we have it).

What the student sees, in words
  • Link still valid → tapped → straight in (dashboard, or onboarding on a first login).
  • Link expired (past 60 min) → friendly "expired, here's a fresh one" banner + resend + OTP.
  • Link already used once → same friendly banner (single-use is a feature, not an error).
  • Didn't arrive → "Didn't get it? Resend · or use a code" is on the check-inbox screen.
  • Email trouble entirely → OTP is always one tap away as the backup.

Why + is it safe

  • Why: the code copy-paste + app↔inbox flip is where first-time students (and parents) drop off before they start. One tap removes it.
  • Just as safe: a login link and an OTP prove the same one thing — that you control your email. The link is the tap-friendly version of the code.
  • Actually a tighter window: single-useexpires in 60 minAPP-key signed — consumed on the first tap, so a forwarded link can't be reused.
  • Children unchanged: guardian consent still gates every bit of a child's data — the link only gets them to the app, exactly like the OTP does today.
  • Nobody's stuck: OTP stays visible as the backup, and any expired/used link falls back to it automatically. A flag lets us flip primary back to OTP instantly if ever needed.
