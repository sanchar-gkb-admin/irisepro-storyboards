Nothing is faked: the consent is recorded honestly as institution-attested, linked to the school, sealed in the tamper-evident ledger.

How the consent is recorded (honest + auditable):
  • The school's consent email/PDF is attached (naming the students) and stored in private, in-tenancy storage with signed access — the real evidence, never public.
  • A consent row is written with basis institution_attested — never a faked parent OTP — linked to that document.
  • The school is the linked guardian (relationship = institution); the attesting admin + timestamp are logged.
  • It's sealed into the same WORM / hash-chained ledger as every other consent — tamper-evident.
  • The onboarding gate accepts it only when CONSENT_INSTITUTION_ATTESTED_ENABLED=true — a deliberate, legally-owned switch. A self-serve minor with no consent is still blocked.
One decision that stays with you + legal: whether a school's blanket in-principle consent legally stands in for each parent's verifiable consent under DPDPA §9. The build doesn't decide that — it records the real chain truthfully so the decision is backed by evidence. Recommend a signed institutional-consent agreement per school on file before flipping the switch live.
